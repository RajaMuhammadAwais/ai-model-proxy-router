You are a senior principal software architect + backend engineer + DevOps engineer.

Your task is to design and build a production-grade “AI Model Proxy & Intelligent Router System” that acts as a universal OpenAI-compatible gateway for multiple LLM providers.

This system MUST be:

- Fully dynamic (NO hardcoded logic, NO fixed routing rules written manually)
- Research-based (use real provider capabilities and documented APIs)
- Extensible (plugin-based provider architecture)
- Production-ready (logging, retries, rate limiting, caching, auth)
- Clean architecture (separation of concerns)
- Secure by design
- Config-driven (all behavior must come from config or runtime metadata)

---

## 🎯 CORE OBJECTIVE

Build a system that exposes an OpenAI-compatible API:

/v1/chat/completions
/v1/models
/v1/embeddings

But internally routes requests intelligently across multiple LLM providers such as:

- NVIDIA NIM API
- OpenRouter
- Groq
- Gemini API
- Ollama (local models)
- Any additional OpenAI-compatible provider

---

## 🧠 INTELLIGENT ROUTING REQUIREMENTS

Routing MUST NOT be hardcoded.

Instead implement:

### 1. Capability-based routing
Each model must declare metadata:

- context length
- latency score (dynamic measured)
- cost per token
- reasoning strength
- coding strength
- tool-calling support
- multimodal support

This metadata must be:
- loaded from config OR provider introspection
- updated dynamically (not static constants)

---

### 2. Request Analyzer (NO hardcoded rules)
Build a prompt classifier module that:
- detects intent (coding, reasoning, chat, summarization, etc.)
- uses lightweight ML or embedding similarity OR LLM-based classification
- avoids static if/else rules

---

### 3. Router Engine
Must choose best model based on:
- latency
- cost
- capability match
- current load
- provider health

Use a scoring system that is:
- configurable
- explainable
- dynamic

---

### 4. Fallback System
If provider fails:
- retry with exponential backoff
- switch to next best model automatically
- never crash request

---
